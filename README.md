# 과제 주제 (문제 정의)

**과제명: 자동화 파이프라인 및 RAG 기반 교내 포털 맞춤형 데일리 AI 비서**

- **문제 정의:** 수많은 교내 공지사항, 장학금, 비교과 프로그램, 취업 정보가 매일 학교 포털에 쏟아지지만, 학생들은 바쁜 학업이나 시험 기간 등으로 인해 주기적인 포털 접속을 놓치는 경우가 많습니다. 이로 인해 자신에게 꼭 필요한 혜택과 유익한 정보를 뒤늦게 확인하거나 아예 놓쳐버리는 '정보 비대칭' 및 '기회 상실' 문제가 심각합니다.
    
- **해결 방안:** 파이썬 웹 크롤링을 통해 매일 교내 포털의 최신 정보를 자동 수집하여 Vector DB에 적재하고, 학생의 전공및 관심사를 바탕으로 매일 유익한 정보만 요약해 주는 '데일리 브리핑' 및 '할루시네이션 없는 Q&A' RAG 시스템을 구현합니다.
    

# 주제 선정 배경 (이유 또는 동기)

**1) 포털 미접속으로 인한 혜택 누락 경험** 시험 기간이나 과제가 몰리는 시기에는 거의 1주일 넘게 학교 포털에 들어가지 않는 경우가 많았습니다. 뒤늦게 포털에 접속해 보면 저에게 정말 유용했던 교내 프로그램이나 장학금 신청 기한이 이미 지나버려 허탈했던 경험이 여러 번 있었습니다. 일일이 포털에 접속하지 않아도 나에게 필요한 소식만 배달해 주는 시스템의 필요성을 절실히 느꼈습니다.

**2) 기존 포털의 개인화 부재 및 피로도** 포털에 접속하더라도 타 단과대 공지나 나와 무관한 행사 정보들이 혼재되어 있어, 정작 내게 필요한 정보를 찾는 데 피로감이 큽니다. 매일 올라오는 수십 개의 공지를 AI가 읽고 "오늘 컴퓨터공학과 학생에게 도움 될 만한 소식 3가지"를 필터링해 준다면 학생들의 시간을 획기적으로 절약할 수 있다고 판단했습니다.

**3) 실사용 빈도(Retention)가 높은 일상적 AI 서비스 기획** 특정 기간에만 쓰이는 시스템(예: 수강신청)을 넘어, 학기 중 매일매일 로그인하여 데일리 브리핑을 확인하고 교내 프로그램에 대해 질문할 수 있는 지속 가능한 프로덕트를 만들고 싶었습니다.

# 과제 수행 계획

**Step 1. 목표 서비스 정의 및 크롤링 파이프라인 구축 (3~4월)**

- **데이터 파이프라인 수립:** 교내 포털의 공지사항, 비교과 프로그램, 장학금 게시판 등을 타겟으로 파이썬기반의 웹 크롤러를 구축합니다.
    
- **자동화 설정:** 매일 특정 시간(예: 자정)에 새로운 게시글의 제목, 본문, 신청 기한, 링크를 자동으로 수집하는 파이프라인을 완성합니다.
    

**Step 2. 비정형 데이터 정제 및 Vector DB 적재 (5~6월)**

- **텍스트 정제:** 수집된 공지글의 불필요한 HTML 태그를 제거하고 메타데이터(날짜, 카테고리)를 결합하여 구조화합니다.
    
- **임베딩 및 적재:** 오픈소스 임베딩 모델을 활용하여 정제된 최신 공지 데이터를 Chroma 등의 Vector DB에 매일 업데이트하는 구조를 만듭니다.
    

**Step 3. 개인 맞춤형 요약 및 RAG 백엔드 구현 (7~8월)**

- **맞춤형 필터링 로직:** FastAPI 백엔드에서 사용자의 전공 및 관심 키워드(예: SW, 프로그래밍, 자격증)를 기반으로, Vector DB에 오늘 적재된 데이터 중 연관성 높은 항목만 추출하는 로직을 구현합니다.
    
- **Q&A 추론 파이프라인:** "이번 주 마감인 근로장학금 있어?"와 같은 질문에 외부 검색 없이 교내 데이터만으로 정확한 출처(링크)와 함께 답변하는 RAG 추론 구조를 완성합니다.
    

**Step 4. Streamlit UI 연동 및 프로토타입 테스트 (9~10월)**

- **대시보드 구현:** 교수님께서 제안하신 Streamlit을 활용하여, 접속 시 '오늘의 요약 브리핑'을 보여주고 챗봇과 대화할 수 있는 직관적인 UI를 개발합니다.
    
- **로컬 환경 테스트:** 매일 수집되는 실제 교내 데이터를 바탕으로 할루시네이션(환각) 여부를 1차 검증하고 추천 결과의 질을 테스트합니다.
    

**Step 5. 시스템 안정화 및 최종 점검 (11월)**

- 크롤링 중단 예외 처리, 데이터 중복 적재 방지 등 파이프라인의 안정성을 최종 점검합니다.
    
- 산학협력 프로젝트 최종 성과물 시연 및 결과 보고서를 작성합니다.
    

# 기대효과 (활용방안)

**1) 정보 비대칭 해소 및 학생 혜택 수혜율 극대화**

- 매일 포털을 확인하는 수고를 덜어주며, 바쁜 학생들도 교내 유익한 프로그램과 지원금 등의 혜택을 놓치지 않고 100% 활용할 수 있도록 돕습니다.
    

**2) 매일 사용하는 서비스 구현**

- 일회성 챗봇이 아닌 매일 아침 브리핑을 확인하는 형태의 서비스로서, 향후 교내 공식 앱으로 편입될 수 있을 만큼 높은 실용성과 파급력을 가집니다.
    

**3) Agentic AI로의 확장 기반 마련 (Tool Calling)**

- 현재는 크롤링(수집)과 요약(RAG) 중심이지만, 추후 Agentic 구조를 고도화하여 챗봇 창에서 곧바로 "이 멘토링 프로그램 나 대신 신청해 줘"라고 명령하면 봇이 폼(Form)을 대신 제출해 주는 수준까지 확장할 수 있는 탄탄한 기반이 됩니다.
    

# 6. 데이터 확보 및 활용 계획

본 과제는 교내 퍼블릭 데이터의 자동 수집과 이를 처리하는 파이프라인에 초점을 맞춥니다.

**1) 교내 퍼블릭 데이터 (크롤링 기반 수집)**

- **확보 대상:** 공주대 포털 일반공지, 학사공지, 장학공지, 취업정보 및 각 부서별 주요 프로그램 모집 글.
    
- **확보 방법:** Python 크롤링 스크립트를 통해 주기적(Daily)으로 텍스트, 게시일, URL 등을 스크래핑합니다.
    
- **활용 계획:** 정제 과정을 거쳐 Vector DB에 최신 상태로 유지하며, 챗봇이 답변을 생성할 때 반드시 원문 링크(URL)를 출처로 제공하여 정보의 신뢰도를 100% 보장합니다.
    

**2) 사용자 프로필 데이터 (개인화 연산용)**

- **확보 대상:** 챗봇 사용자의 소속 학과, 학년, 주요 관심 키워드(예: 알고리즘, 단기근로 등).
    
- **활용 계획:** Streamlit UI 최초 진입 시 입력받아 메모리 혹은 경량화된 JSON 형태로 관리하며, 매일 수십 개의 공지 중 해당 학생에게 가장 우선순위가 높은 공지 3~5개를 선별(Filtering)하는 매개변수로만 활용합니다.
