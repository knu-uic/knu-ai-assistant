# 1. 과제 주제 (문제 정의 및 해결 방안)

**과제명: 확장형 RDBMS 웹앱 및 RAG 기반 교내 통합 지능형 학생 비서 플랫폼**

- **문제 정의:** 수많은 교내 공지사항과 혜택이 매일 포털에 쏟아지지만, **나와 무관한 타 단과대 소식이나 불필요한 정보들이 무분별하게 혼재되어 있어 정작 내게 필요한 정보를 찾으려면 숨은그림찾기를 하듯 엄청난 탐색 리소스가 소모**됩니다. 이로 인해 학생들은 바쁜 학업 속에서 정보 확인에 피로감을 느끼고, 결국 꼭 필요한 장학/취업 혜택마저 놓쳐버리는 '정보 비대칭' 문제가 심각합니다. 또한, 전공 학습 시 방대한 수업 자료를 스스로 요약하고 시험을 대비하는 데 막대한 시간이 소모됩니다.
    
- **해결 방안:** 초기부터 단일 챗봇으로 구현 시 발생하는 확장성의 한계를 극복하기 위해, '웹앱(게시판) 기반 데이터 적재 → 챗봇 연동'의 2-Phase 아키텍처를 도입합니다. 파이썬 크롤러와 AI로 정제된 교내 데이터, 그리고 학생이 업로드한 수업 자료를 RDBMS에 체계적으로 적재한 후, 이를 기반으로 구동되는 **노이즈 없는 맞춤형 웹 게시판**과 할루시네이션(환각) 없는 RAG 챗봇을 통합 제공합니다.
    

# 2. 주제 선정 배경 (이유 또는 동기)

1. **기존 포털의 정보 과부하 및 탐색 피로도:** 현재 학교 포털은 모든 학과와 부서의 공지가 필터링 없이 한곳에 섞여 올라옵니다. **나와 전혀 상관없는 수십 개의 글 제목을 일일이 클릭해 보며 진짜 필요한 공지를 걸러내야 했던 피로감이 상당했습니다.** AI가 이를 읽고 "오늘 컴퓨터공학과 3학년에게 꼭 필요한 소식 3가지"만 정확히 큐레이션해 준다면 학생들의 시간과 스트레스를 획기적으로 줄일 수 있습니다.
    
2. **포털 미접속으로 인한 혜택 누락 경험:** 시험 기간 등 바쁜 시기에 위와 같은 탐색 피로도 때문에 포털 접속을 미루다가, 유용한 교내 프로그램이나 장학금 기한을 놓치는 학생들의 Pain Point를 직접 겪으며 능동적인 맞춤형 정보 배달 서비스의 필요성을 절실히 느꼈습니다.
    
3. **능동적 학습 보조 도구의 부재:** 단순히 정보를 검색하는 것을 넘어, 전공 PDF 자료를 올리면 AI가 즉각적으로 예상 문제를 만들어주는 기능을 더해, 시험 기간에도 매일 접속하게 되는 높은 리텐션(Retention)의 프로덕트를 기획했습니다.
    
4. **시스템 확장성을 고려한 아키텍처 설계:** 챗봇 단일 인터페이스의 한계를 벗어나, 관계형 데이터베이스(RDBMS)에 정형 데이터를 선행 적재함으로써 향후 교내 공식 앱이나 다른 웹 서비스로 유연하게 확장(API 제공 등)할 수 있는 구조를 증명하고자 합니다.
    

# 3. 핵심 기능 및 과제 수행 계획 (Phasing)

**[핵심 개발 우선순위]**

- **1순위: 스마트 공지사항 보드 (데이터 수집 및 분류)**
    
    - 기능: 교내 포털 자동 크롤링 및 카테고리별(장학, 취업 등) 분류 후 웹앱 게시판 제공.
        
- **2순위: 통합 지능형 챗봇 (데이터 기반 Q&A)**
    
    - 기능: RDBMS에 축적된 검증된 교내/학습 데이터만을 활용해 할루시네이션 없는 정확한 출처 기반 Q&A 제공.
        
- **3순위: AI 학습 보조 게시판 (문제 자동 생성)**
    
    - 기능: 챗봇에 수업 자료(PDF 등) 첨부 시, AI가 핵심 내용을 추출하여 예상 문제를 생성하고 과목별 게시판에 아카이빙.
        

**[단계별 개발 로드맵]**

- **Phase 1. 데이터 파이프라인 및 웹앱 뼈대 구축 (3~5월)**
    
    - 웹 크롤러 구축 및 AI(LLM)를 활용한 카테고리/추천 대상 정제 자동화 파이프라인 개발.
        
    - 확장성을 고려한 RDBMS 테이블(ERD) 설계 및 데이터 적재 테스트.
        
    - FastAPI와 Streamlit을 활용하여 데이터를 시각화하는 기본적인 게시판 형태의 웹앱 UI 구현.
        
- **Phase 2. RAG 챗봇 고도화 및 연동 (6~8월)**
    
    - RDBMS에 적재된 교내 공지사항 데이터를 토대로 정확한 답변을 제공하는 RAG 챗봇 백엔드 구현.
        
    - 사용자가 업로드한 수업 자료(PDF)를 분석하여 예상 문제를 생성하는 프롬프트 엔지니어링 및 추출 로직 구현.
        
- **Phase 3. 문제은행 게시판 확장 및 라우팅 에이전트 도입 (9~11월)**
    
    - 생성된 문제와 정답을 RDBMS '과목별 문제 게시판' 테이블에 적재 및 UI 연동.
        
    - 사용자의 질문 의도를 파악하여 RDBMS의 어느 테이블(공지사항 vs 문제은행)을 조회할지 지능적으로 결정하는 LangGraph 기반 라우팅 에이전트(Routing Agent) 도입.
        
    - 시스템 안정화 테스트 및 산학협력 프로젝트 최종 성과물 시연.
        

# 4. 기대효과 (활용방안)

1. **학생 관점 (탐색 피로도 해소 및 수혜율 극대화):** 나와 무관한 정보들을 일일이 열어봐야 했던 **기존 포털의 탐색 피로도를 완전히 해소합니다.** 매일 알아서 배달되는 맞춤형 브리핑을 통해 수고를 덜고 교내 혜택 수혜율을 높입니다.
    
2. **학습 관점 (집단 지성을 통한 효율 극대화):** AI 문제은행을 통해 전공 학습 시간을 단축할 뿐만 아니라, 동기들과 문제를 공유하는 게시판을 통해 동일 과목 수강생 전체의 학습 퀄리티를 동반 상승시키는 시너지를 냅니다.
    
3. **아키텍처 관점 (안정성 및 확장성):** 데이터를 챗봇에 직접 종속시키지 않고 RDBMS와 웹앱 형태로 분리 및 선행 구축함으로써, 향후 학교 시스템이나 타 서비스와의 API 연동이 매우 용이해집니다.
    
4. **서비스 관점 (높은 실사용 빈도):** 정보 확인(평시)부터 문제 풀이(시험 기간)까지 학생의 모든 학사 사이클을 커버하여 매일 사용하는 교내 필수 플랫폼으로 자리 잡을 수 있습니다.
    

# 5. 데이터 확보 및 활용 계획

**1) 교내 퍼블릭 데이터 (크롤링 기반 수집)**

- **확보 대상:** 공주대 포털 일반/학사/장학/취업 공지 및 주요 프로그램 글.
    
- **활용 계획:** AI로 메타데이터(추천 대상, 마감일 등)를 정제하여 RDBMS 테이블에 적재. 챗봇 답변 시 반드시 RDBMS 내의 원문 링크(URL)를 출처로 제공하여 신뢰도 보장.
    

**2) 학생 학습 데이터 (User Upload)**

- **확보 대상:** 사용자가 챗봇을 통해 업로드하는 특정 과목의 수업 자료(PDF, 텍스트).
    
- **활용 계획:** LLM을 통해 지식 기반 Q&A 데이터로 변환 후, RDBMS의 `Quiz_Archive` 테이블에 저장하여 해당 과목을 수강하는 다른 학생들도 열람할 수 있는 집단 지성 게시판으로 활용.
    

**3) 사용자 프로필 데이터 (개인화 연산용)**

- **확보 대상:** 소속 학과, 학년, 주요 관심 키워드.
    
- **활용 계획:** 웹앱 접속 시 입력받아 관리하며, RDBMS에 적재된 수십 개의 데이터 중 해당 학생에게 가장 우선순위가 높은 정보를 선별(SQL Filtering)하는 핵심 매개변수로 활용.
