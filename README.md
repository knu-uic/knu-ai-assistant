# PDF OCR Text Extractor
`pdf2image`를 사용하여 PDF 페이지를 이미지로 변환하고, `easyocr`을 활용하여 이미지 내의 한국어 및 영어 텍스트를 추출 가능

## 주요 라이브러리 및 코드 설명

### 1. 주요 라이브러리
* **easyocr**: 딥러닝 기반의 OCR 엔진으로, 이미지에서 글자를 읽어냅니다.
* **pdf2image**: PDF 파일을 이미지 파일(PIL 객체)로 변환해주는 도구입니다.
* **numpy**: 이미지 객체를 수학적 연산이 가능한 '행렬(Array)' 형태로 변환하여 OCR 엔진이 처리할 수 있게 돕습니다.

### 2. 코드 동작 흐름
1.  **OCR 엔진 초기화**: `easyocr.Reader(['ko', 'en'])`를 통해 한글(ko)과 영어(en)를 인식할 준비를 합니다.
2.  **PDF -> 이미지 변환**: `convert_from_path` 함수가 PDF의 각 페이지를 렌더링하여 **PIL 이미지 객체 리스트**를 생성합니다.
3.  **넘파이 배열 변환**: `np.array(image)`를 통해 이미지 객체의 껍데기를 벗기고, 내부의 **RGB 픽셀 데이터(숫자)**를 행렬 구조로 펼칩니다.
    •	껍데기 벗기기: image 객체(PIL)는 메타데이터와 기능이 포함된 '상자'와 같습니다. 넘파이에 넣는 순간 이 상자는 버려지고 내부의 순수한 RGB 숫자 데이터만 추출됩니다.
    •	배열 인터페이스(Array Interface): 이미지 객체 내부에는 **__array_interface__**라는 약속된 통로가 설계되어 있습니다.
    •   넘파이가 이 객체를 만나면, 이 인터페이스를 통해 메타데이터를 행렬의 구조로서 사용하고 픽셀 바이너리를 숫자 데이터로 변환하겠다라는 약속을 자동으로 수행합니다.
    •   결과적으로 데이터의 내용은 변하지 않지만, 컴퓨터(AI)가 즉시 수학 연산을 할 수 있도록 [세로, 가로, RGB] 형태의 3차원 숫자 격자판으로 재배열되는 것입니다.
4.  **텍스트 추출**: `reader.readtext`가 넘파이 행렬을 분석하여 글자를 찾아내고, 이를 리스트에 담아 반환합니다.

## 실행 주의 사항
1. 시스템에 `poppler`가 설치되어 있어야 `pdf2image`가 작동합니다.
